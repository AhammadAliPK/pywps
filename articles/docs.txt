            <a name="requirements"><h3>Requirements</h3></a>
            <p>
                Current version of PyWPS does require following programms/libs:
                <ul>
                    <li>Web Server (e.g. <a href="http://httpd.apache.org">Apache</a>)</li>
                    <li><a href="http://python.org">Python programming language</a></li>

                    <li><a href="http://pyxml.sourceforge.net/">Python-xml package</a></li>
                </ul>
            </p>
            <p>
                Suggested programs:
                <ul>
                    <li><a href="http://grass.itc.it">GRASS GIS</a></li>
                    <li><a href="http://proj.maptools.org/">PROJ.4</a></li>

                    <li><a href="http://remotesensing.org/gdal/">GDAL/OGR</a></li>
                    <li><a href="http://r-project.org">R</a></li>
                </ul>
            </p>

            <hr />
            <a name="install"><h3>Installation</h3></a>
            <p>

                Before installing, read <a href="https://wald.intevation.org/plugins/scmsvn/viewcvs.php/trunk/INSTALL?rev=1&root=pywps&view=markup">INSTALL</a> file.
            </p>
            <p>
                <ol>
                    <li> You can <a href="/download/">download</a> PyWPS as DEB, RPM or TAR.GZ package. For <code>tar.gz</code>, unpac the package to server's code <code>cgi-bin</code> directory.</li>

                    <li> Adjust your settings in <code>etc/settings.py</code> and eventually <code>etc/grass.py</cocde> files.</li>
                    <li> Point your browser to <code>wps.py</code> file (e.g. <code><a href="http://localhost/cgi-bin/wps.py">http://localhost/cgi-bin/wps.py</a></code>), and you should get something like this:
                        <pre>

&lt;?xml version="1.0" ?&gt;
&lt;ExceptionReport version="1.0.0" xmlns="http://www.opengis.net/ows" 
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"&gt;
        &lt;Exception exceptionCode="MissingParameterValue" locator="request"/&gt;
&lt;/ExceptionReport&gt;
                        </pre>
                    </li>
                </ol>

            </p>

            <p>
                <strong>Troubleshooting</strong> <br />
    If you got some other message, like e.g.:
     

    <pre>
Traceback (most recent call last):
  File "trunk/index.py", line 53, in ?
    from Wps import wpsexceptions
  File "/home/jachym/prog/pywps/trunk/Wps/wpsexceptions.py", line 8, in ?
    from xml.dom.minidom import Document
ImportError: No module named xml.dom.minidom
</pre>

     
    Than something is wrong with your Python installation or with the program.
    This message means, that the <code>xml.dom.minidom</code> package is not installed in
    your system.
            </p>
